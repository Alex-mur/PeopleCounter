# PeopleCounter_WorkerYOLO26 🚦

> Веб-сервис для подсчета людей в зонах интереса на основе видеопотоков. 
> Является частью системы подсчёта посетителей.

---

## 🌟 Возможности

- 🎥 **Multistreaming:** Одновременная обработка нескольких видеопотоков (IP-камеры, RTSP, файл).
- 📦 **Гибкая конфигурация:** Определение зон подсчета через линии (`lines`) и группировка данных (`groups`).
- 💾 **Умное хранение:** Использование `DuckDB` для высокой скорости запросов и аналитики.
- 🧹 **Автоматическая очистка:** Настройка удаления старых логов и данных о проходах (по умолчанию: 7 дней).
- 🔒 **API Безопасность:** Поддержка API-ключей для защиты эндпоинтов.
- 🔄 **Самовосстановление:** Автоматический перезапуск счетчиков при сбоях подключения к потоку.
- 📊 **Агрегация:** Поддержка статистики за сутки, неделю, месяц или в реальном времени (текущий час).

## ⚙️ Настройка

### 1. Файл конфигурации (`config.ini`)

Файл создается автоматически при первом запуске. Вы можете отредактировать его:

```ini
[SERVICE]
port = 8000
api_key = secretkey123
keep_logs_days = 7
```

### 2. Формат данных линий (`lines`)

Настройте линии подсчета в запросах API. Линии определяются координатами `[start_x, start_y]` и `[end_x, end_y]` на изображении.

**Важно:**
- Группы (`groups`) должны быть определены до добавления линий.
- В запросе к линии `group_id` должен соответствовать ID существующей группы.

**Пример JSON для тела запроса:**

```json
{
  "id": 1,
  "name": "Входная группа",
  "description": "Подавочная зона",
  "url": "rtsp://user:pass@camera_ip/stream",
  "vid_stride": 2,
  "lines": [
    {
      "id": 1,
      "name": "Линия 1",
      "start": [100, 300],
      "end": [500, 300],
      "group_id": 1
    }
  ],
  "groups": [
    {
      "id": 1,
      "description": "Посетители"
    }
  ]
}
```

## 📡 API Эндпоинты

Все запросы требуют заголовка `X-Api-Key: secretkey123` (или через параметр `api_key`).

### 📊 `GET /api/info`
Получить информацию о сервисе.
```bash
curl http://localhost:8000/api/info
```
**Ответ:**
```json
{
  "name": "PeopleCounterWorker (YOLO26)",
  "version": "1.0.1",
  "started_at": "2023-10-27T10:00:00"
}
```

### ➕ `POST /api/counters`
Создать новый счетчик и начать работу с потоком.

### 🔄 `PUT /api/counters/{counter_id}`
Обновить настройки существующего счетчика.
> **Примечание:** Сервер перезапустит детекцию для обновления линий/групп.

### 📋 `GET /api/counters`
Получить список всех активных счетчиков и их статус (`running`).

### 🗑 `DELETE /api/counters/{counter_id}`
Остановить счетчик и удалить его запись.

### 🎥 `GET /api/counters/{counter_id}/stream`
Получить MJPEG стрим для отладки или интеграции с видеосистемами.
```bash
curl -O http://localhost:8000/api/counters/1/stream
```

### 📈 `GET /api/counters/{counter_id}/stats`
Получить статистику проходов.
- **Query Params:** `period` (month, week, day, hour).
- Возвращает историю агрегированных данных и данные за текущий час.

### 📜 `GET /api/logs`
Получить системные логи сервиса за указанный период.

## 🐍 Архитектура (Python)

Сервис построен на основе:
- **FastAPI**: Для асинхронной обработки запросов.
- **DuckDB**: Локальная встраиваемая БД для высокой производительности аналитики.
- **Threading**: Использование процессов (демонов) для сбора статистики, очистки данных и перезапуска счетчиков, чтобы не блокировать основной поток API.

### Жизненный цикл приложения (`lifespan`)
При старте сервиса запускаются фоновые потоки:
1.  **Stats Collector**: Каждые 60 сек забирает данные из модуля подсчета, вычисляет `delta` (проходы с момента последнего запроса) и пишет в таблицу `passes`.
2.  **Restarter**: Проверка состояния счетчиков каждые 60 сек. При остановке потоком запускает новый поток.
3.  **Cleanup**: Каждые 60 сек удаляет логи старше `KEEP_LOGS_DAYS` и старые данные `passes` старше 90 дней.

## 📝 Примеры использования

### Отправка новой камеры (cURL)
```bash
curl -X POST http://localhost:8000/api/counters \
  -H "X-Api-Key: secretkey123" \
  -H "Content-Type: application/json" \
  -d '{
    "id": 2,
    "name": "Лобби офиса",
    "url": "http://192.168.1.100:8080/video",
    "vid_stride": 1,
    "groups": [{"id": 1, "description": "Гости"}],
    "lines": [{"id": 1, "name": "Вход", "start": [0,0], "end": [1280, 1280], "group_id": 1}]
  }'
```

### Получение статистики (Python)
```python
import requests
import datetime

url = "http://localhost:8000/api/counters/1/stats"
api_key = "secretkey123"

response = requests.get(url, headers={"X-Api-Key": api_key}, params={"period": "day"})
data = response.json()

print(f"Пройдено за сегодня: {data['history']}")
print(f"Актуально сейчас: {data['realtime_current_hour']}")
```

## ⚠️ Ограничения и рекомендации

1.  **Производительность:** `PeopleCounter` в фоновом потоке зависит от мощности CPU/GPU. При большом количестве камер (`counters`) рекомендуется распределять нагрузку на разные узлы.
2.  **Координаты:** Координаты линий всегда интерпретируются относительно разрешения видео на входе. При изменении разрешения (`width/height`) потоков, координаты линий могут потребовать обновления.
3.  **Точность:** Точность зависит от модели YOLO и качества видеопотока.
